## ⚙️ Установка и настройка

### Шаг 1: Создание Google Таблицы

1. Создайте новую Google Таблицу
2. Скопируйте её ID из URL (между `/d/` и `/edit`)
3. Сохраните ID для следующего шага

### Шаг 2: Настройка Google Apps Script

1. В Google Таблице перейдите в **Расширения** → **Apps Script**
2. Скопируйте содержимое файла `Code.gs` в редактор
3. Обновите константы в начале файла:

```javascript
const TOKEN = "YOUR_BOT_TOKEN_HERE";      // Токен из @BotFather
const SHEET_ID = "YOUR_SHEET_ID_HERE";     // ID вашей таблицы
const GROUP_ID = -1001234567890;           // ID вашей группы
const MINI_APP_URL = "YOUR_WEB_APP_URL";   // Будет после деплоя
const COOLDOWN_DAYS = 30;                  // Кулдаун после отказа
```

### Шаг 3: Деплой веб-приложения

1. В редакторе Apps Script нажмите **Развертывание** → **Новое развертывание**
2. Выберите тип: **Веб-приложение**
3. Описание: `v1`
4. Выполнять как: **От имени владельца**
5. У кого есть доступ: **Все**
6. Нажмите **Развернуть**
7. Скопируйте URL веб-приложения
8. Вставьте его в константу `MINI_APP_URL` в `Code.gs`

### Шаг 4: Настройка Telegram бота

1. Создайте бота через [@BotFather](https://t.me/BotFather)
2. Получите токен и вставьте в `TOKEN`
3. Добавьте бота в вашу группу как администратора с правами:
   - ✉️ Отправка сообщений
   - 👥 Управление участниками
   - 🔇 Ограничение прав

### Шаг 5: Настройка Webhook

1. Откройте Google Таблицу
2. После перезагрузки появится меню **🤖 Bot Admin**
3. Нажмите **Обновить Webhook**
4. Подтвердите успешное подключение

### Шаг 6: Инициализация таблицы

1. В меню **🤖 Bot Admin** нажмите **Init**
2. Будут созданы заголовки колонок

## 📱 Использование

### Для пользователя

1. **Вступление в группу**
   - Пользователь добавляется в группу
   - Бот автоматически ограничивает права на отправку сообщений
   - Отправляется приветственное сообщение с кнопкой «Верификация»

2. **Прохождение верификации**
   - Пользователь нажимает кнопку «Верификация»
   - Открывается Mini App с анкетой
   - Заполняются поля: статус, ферма, оператор, сезоны
   - После отправки данные сохраняются в таблицу
   - Права на отправку сообщений восстанавливаются

3. **Если пользователь еще не был в Британии**
   - Нажимает кнопку «Я еще не был в Британии»
   - Верификация отклоняется
   - Устанавливается кулдаун 30 дней

### Для администратора

#### Меню «🤖 Bot Admin»

| Команда | Описание |
|---------|----------|
| **Init** | Создает заголовки колонок в листе Users |
| **Статистика** | Показывает количество верификаций (✅/❌) |
| **Рассылка** | Отправляет сообщение всем верифицированным пользователям |
| **Обновить Webhook** | Принудительно обновляет webhook бота |

#### Структура таблицы «Users»

| Колонка | Описание |
|---------|----------|
| Date | Дата верификации |
| User ID | Telegram ID пользователя |
| First Name | Имя |
| Last Name | Фамилия |
| Username | Юзернейм |
| Farm | Название фермы |
| Operator | Агентство-оператор |
| Seasons | Количество сезонов |
| Status | ✅ OK / ❌ No |
| Notes | Примечания |

## 🔒 Защита и ограничения

### Дедупликация запросов
- Используется `CacheService` для предотвращения двойной обработки
- Кэш хранится 60 секунд

### Кулдаун система
- После неудачной верификации («Еще не был в Британии»)
- Пользователь не может пройти верификацию 30 дней
- Настройка: `COOLDOWN_DAYS` в `Code.gs`

### Тайм-ауты
- Код разделен на мелкие функции
- Предотвращение превышения лимита Google (30 секунд)

## 🔄 Обновление кода

1. Внесите изменения в `Code.gs` или `index.html`
2. В редакторе Apps Script: **Развертывание** → **Управление развертываниями**
3. Нажмите ✏️ рядом с активным развертыванием
4. Измените описание на новую версию (например, `v2`)
5. Нажмите **Развернуть**
6. Если изменился URL — обновите `MINI_APP_URL` в `Code.gs`
7. Выполните **Обновить Webhook** из меню таблицы

## 🐛 Решение проблем

### Бот не реагирует на новые сообщения
1. Проверьте, что webhook установлен: **Обновить Webhook**
2. Убедитесь, что токен правильный
3. Проверьте логи в Apps Script (**Выполнения**)

### Пользователи не могут пройти верификацию
1. Проверьте, что `MINI_APP_URL` актуален
2. Убедитесь, что веб-приложение доступно всем
3. Проверьте права бота в группе

### Ошибки при сохранении в таблицу
1. Убедитесь, что `SHEET_ID` правильный
2. Проверьте наличие листа «Users»
3. Выполните **Init** для создания заголовков
